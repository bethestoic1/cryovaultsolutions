How Multi-Accounts Actually Works

This isn't just a UI trick where you filter the same addresses into different views.

Each account is genuinely independent:

  • Separate addresses on every network. Account 1 has a different Ethereum address than Account 2. Your DeFi approvals go to Account 2's address, not your hodl address. If Account 2 gets rugged, Account 1 is untouched.
  • Separate transaction history. Each account shows only its own transactions. No clutter. No confusion.
  • Separate balances. You see the total balance across all accounts, but each account shows only its holdings. Crystal-clear portfolio segmentation.
  • All managed under one seed phrase. This is the magic part. Mathematically, Tangem derives each account from the same master seed using BIP-44 standards (the same standard every hardware wallet uses). You backup once. You recover everything.

The Archive System: Clean Up Without Losing Anything

You want to create 20 accounts, but you'll eventually hit the limit. Tangem's archive system solves this without deleting anything:

Archive (Not Delete)

When you archive an account, it disappears from your main view and your total balance calculation. But all funds, all addresses, all transaction history stay intact.

Restore Anytime

Un-archive an old account whenever you want. All your history comes back. No loss of data. It's like storing a file in a hidden folder.

Portability: BIP-44 Standard = Recovery Anywhere

Here's what makes this genuinely self-custody:

Tangem uses BIP-44 standards for account derivation. This is the same standard Bitcoin, Ethereum, and every major blockchain uses. It means:

  • Your accounts are recoverable on any wallet software. If Tangem disappears tomorrow, you can import your seed phrase into Ledger, Trezor, MetaMask, or any BIP-44 compatible wallet. All 20 accounts appear. All your funds appear. No lock-in.
  • Accounts are mathematically derived from your seed. Not stored on Tangem's servers. Not proprietary. Pure cryptography.
  • Custom tokens auto-route to the correct account. If you add a custom token to your DeFi account, Tangem's wallet automatically recognizes the derivation path and moves it there. No manual sorting.

The Math (If You Care)

Tangem uses Hierarchical Deterministic wallets (BIP-32), which means it generates billions of unique keys from a single master seed using cryptographic derivation paths. Each account gets its own branch of this tree (m/44'/60'/0'/0/0 for Account 1, m/44'/60'/0'/0/1 for Account 2, etc.). Same seed. Different branches. Mathematically unique and secure.

TL;DR: One seed, infinite accounts, zero lock-in.
